Features

Everything your catalog, nothing you don't need.

Skill Browser reads your .claude/ tree and plugin marketplace locally, then gives you six ways to work with it.

🔍

Browse all sources

Plugins, user-global, and project-local skills in one unified filterable list.

⚙

Compose mode

Describe intent in plain language; get ranked matches with confidence scores.

📜

Recipes

Save skill chains as reusable, share-ready recipes — version-controllable.

🛒

Basket & prompt generation

Drop skills into a basket, then export a paste-ready prompt for any workflow.

🔒

Local-only, no API calls

Runs entirely in your browser against a static index. No network, no telemetry.

⚡

Agent-friendly CLI

intent.mjs and query.mjs let agents introspect your catalog programmatically.

How it works

Three steps, under a minute.

No build step, no package manager, no config. Drop in, launch, compose.

1

Install

Clone (or copy) the folder into ~/.claude/ecc-browser/. Node 18+ is the only prerequisite.

2

Launch

Run ./launch.sh (or open index.html). The indexer builds a local catalog in seconds.

3

Compose

Type a goal, review ranked matches, drop the best into your basket, copy the prompt.

CLI preview

Same catalog, scriptable from the shell.

intent.mjs takes a plain-language goal and returns ranked skills as JSON or pretty output.

~/.claude/ecc-browser 
$ node intent.mjs "review my code for security"

matched 12 skills across 8 sources (38ms)

For agents

Your agents get the same menu you do.

Claude Code agents invoke Skill Browser through the ecc-menu skill — the exact same index powers both the UI and the programmatic interface.

One index, two front-ends.

Humans open index.html. Agents call intent.mjs or query.mjs. Both read the same data.json catalog — so suggestions stay consistent whether you're browsing manually or delegating to a sub-agent.

This means agents can self-discover capabilities ("what skills exist for Postgres migrations?") without hard-coded lists, and users can share recipes with agents by name.

[agent] "find me skills for load-testing"
→ ecc-menu skill
 → query.mjs --intent ...
  → data.json (local)
   ← ranked JSON results
  ← formatted suggestions
 ← agent invokes top match
